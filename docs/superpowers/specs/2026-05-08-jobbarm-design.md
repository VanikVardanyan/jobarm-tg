# JobArm — TG Mini App Design Spec

**Date:** 2026-05-08
**Status:** Approved

---

## 1. Идея и цель

Маркетплейс разовых работ в Армении в виде Telegram Mini App. Заказчик публикует заявку — мастера откликаются. MVP для проверки гипотезы: запуск рекламы в FB/Instagram → замер конверсии.

**Домен:** jobArm

---

## 2. Пользователи

Один аккаунт может быть одновременно и мастером и заказчиком. Роли не исключают друг друга.

| Роль | Описание |
|---|---|
| Заказчик | Публикует заявки, выбирает мастеров |
| Мастер | Откликается на заявки, выполняет работу |

---

## 3. Стек

| Компонент | Выбор |
|---|---|
| Frontend | React + Vite + Telegram WebApp SDK |
| UI | shadcn/ui + Tailwind CSS |
| Тема | light/dark через `window.Telegram.WebApp.colorScheme` |
| Шрифт | Noto Sans (латиница + кириллица + армянский) |
| Backend | Node.js + Fastify + Prisma |
| Bot | Telegraf |
| БД | PostgreSQL |
| Кэш/очередь | Redis |
| Языки | RU + EN |
| Dev окружение | Docker Compose + ngrok |
| Монорепо | pnpm workspaces |

### Структура репозитория

```
/apps
  /web        ← React Mini App (Vite)
  /server     ← Fastify API + Telegraf bot
/packages
  /shared     ← общие TypeScript типы
docker-compose.yml
```

---

## 4. Точка входа

```
Реклама FB/Instagram
  → кнопка "Перейти" → t.me/jobarmbbot?start=ref
    → Telegram открывается → бот: приветствие + кнопка "Открыть JobArm"
      → Mini App запускается
        → Онбординг внутри приложения
```

Бот после онбординга — только нотификатор. Вся логика внутри Mini App.

---

## 5. Онбординг (первый вход)

1. Выбор языка: **RU / EN**
2. Выбор роли: **[Ищу мастера]** | **[Я мастер]** (вторую роль можно добавить позже в профиле)
3. Форма: имя + номер телефона
4. Если мастер — выбор категорий (несколько)
5. Главный экран

---

## 6. Навигация

**4 таба внизу:** Главная · Мастера · Уведомления · Профиль

**Главная** содержит переключатель вверху: **Заказчик / Мастер**

---

## 7. Экраны

### 7.1 Главная / Заказчик

- Переключатель Заказчик | Мастер вверху
- Кнопка "+ Новая заявка"
- Список своих заявок: название, бюджет, даты, статус (кол-во откликов / ждём)
- Тап на заявку → список откликнувшихся мастеров

### 7.2 Создать заявку

Поля:
- Категория (обязательно)
- Описание задачи (обязательно)
- Бюджет в AMD
- Даты: от / до

Кнопка "Опубликовать" → заявка уходит всем мастерам категории через бот-уведомление.

### 7.3 Список откликов

- Карточка каждого мастера: имя, рейтинг ★, категории, кол-во выполненных заказов, короткий комментарий
- Кнопка "Выбрать → открыть номер"
- После выбора: номер телефона мастера открывается, заказ переходит в статус "в работе"

### 7.4 Главная / Мастер

- Переключатель Заказчик | Мастер вверху
- Секция "Новые заявки": карточки с описанием + кнопка **"Откликнуться"** прямо на карточке
- Секция "Мои заказы": активные заказы со статусом + кнопка "Завершить"

### 7.5 Каталог мастеров (таб "Мастера")

- Фильтр по категориям: горизонтальные чипы со скроллом, правый край с градиентом-затуханием (намёк на скролл)
- Карточки мастеров: имя, рейтинг, категории, кол-во заказов
- Тап → профиль мастера

### 7.6 Профиль мастера

- Имя, рейтинг ★, кол-во выполненных заказов
- Категории услуг
- О себе
- Отзывы (список)
- Кнопка "Связаться" → открывает номер телефона (прямой контакт без заявки)

### 7.7 Профиль пользователя (таб "Профиль")

- Имя, телефон
- Управление ролями (добавить роль мастера / заказчика)
- Категории (если мастер)
- Смена языка

---

## 8. Жизненный цикл заказа

```
новая
  → мастер откликается → [несколько откликов возможны]
    → заказчик выбирает мастера → в работе (номер открыт)
      → мастер нажимает "Готово"
        → заказчик подтверждает "Готово"
          → завершена → форма отзыва для заказчика
```

**Статусы:** `new` → `in_progress` → `pending_confirmation` → `completed`

Оба участника подтверждают завершение. Только после этого открывается форма отзыва.

---

## 9. Уведомления (через бот)

| Событие | Получатель |
|---|---|
| Новая заявка в категории | Все мастера категории |
| Новый отклик на заявку | Заказчик |
| Заказчик выбрал тебя | Мастер |
| Мастер отметил "Готово" | Заказчик |
| Заказчик подтвердил завершение | Мастер |
| Новый отзыв на профиле | Мастер |

---

## 10. Рейтинг и отзывы

- Оценка: 1–5 звёзд
- Текстовый отзыв (опционально)
- Отзыв оставляет заказчик после подтверждения завершения
- Рейтинг мастера = среднее по всем отзывам
- Отображается везде: каталог, список откликов, профиль

---

## 11. Раскрытие контакта

Номер телефона мастера скрыт до момента выбора. Заказчик видит профиль (услуги, рейтинг, отзывы) → нажимает "Выбрать" → открывается номер. Это защищает мастеров от спама и стимулирует заказчика к осознанному выбору.

---

## 12. Dev окружение

```yaml
# docker-compose.yml
services:
  postgres:   # PostgreSQL 16
  redis:      # Redis 7
  api:        # Fastify + Prisma
  bot:        # Telegraf (применяет миграции при старте)
```

Mini App (Vite) запускается локально. ngrok пробрасывает порт для тестирования в Telegram.

---

## 13. Вне скоупа MVP

- Оплата внутри приложения
- Геофильтрация по городам
- Фото в профиле
- Армянский язык (HY) — шрифт подключён, локализация позже
- Верификация мастеров
- Монетизация (платная подписка / комиссия)
- Веб-версия
